# Release Note: V1.3.3

Date: 2026-03-28

## Release Intent

Align generated backend CRUD architecture with clean separation of concerns so projects generated by `admin:setup` are maintainable by default.

## What Changed

`admin:setup` now generates and wires:

- `app/Http/Controllers/AdminDashboardCrudController.php`
- `app/Support/AdminDashboard/FieldResolver.php`
- `app/Support/AdminDashboard/EntityTableResolver.php`
- `app/Support/AdminDashboard/CrudPayloadBuilder.php`

The command also ensures CRUD routes under `/api/admin-dashboard/*`, including entities, schema, records, and user admin actions.

## Architectural Outcome

- Controller orchestrates request flow.
- `FieldResolver` handles UI field metadata rules.
- `EntityTableResolver` handles table resolution and metadata lookup.
- `CrudPayloadBuilder` handles payload whitelisting and FK validation.

## Verification Checklist

- Package update installed in sandbox via Composer.
- `php artisan admin:setup --force --no-interaction` executed successfully.
- Generated support directory contains multiple files (3 services).
- `php -l` passes for generated controller and support classes.
- CRUD routes present in `routes/api.php`.
- `php artisan test` baseline passes.

## Upgrade Guidance

1. Update package in project:

```bash
composer update elmekadem/architector-admin --with-all-dependencies
```

2. Regenerate setup artifacts:

```bash
php artisan admin:setup --force
```

3. Validate generated files and routes.
